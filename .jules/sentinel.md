## 2024-05-18 - XSS in Dynamic DOM Injection via innerHTML
**Vulnerability:** The application was fetching external JSON and CSV data and injecting it directly into the DOM using `innerHTML` without sanitization. This allowed for Cross-Site Scripting (XSS) if the remote files were ever compromised or contained malicious payloads. URLs in `href` attributes were also not sanitized against dangerous protocols like `javascript:`.
**Learning:** Using `innerHTML` with unsanitized dynamic data is a critical vulnerability vector. When writing security helpers like `escapeHTML`, inputs must be explicitly cast to strings to prevent array bypass vulnerabilities. For URLs, simple prefix checks aren't enough; the `URL` API's `.protocol` property must be used to reliably identify and block dangerous schemes like `javascript:`, `data:`, and `vbscript:`.
**Prevention:** Always sanitize dynamic data before injecting it into the DOM. Prefer safer alternatives like `textContent` where HTML parsing is not required. When `innerHTML` is necessary, use a robust sanitization library or custom helper that enforces string conversion and escapes HTML entities (`&`, `<`, `>`, `"`, `'`). Sanitize dynamic URLs by validating the `protocol` property using the `URL` API.
