## 2025-04-11 - False Positive Security Audits Breaking Production
**Vulnerability:** An AI security auditor identified correctly installed dependency packages (`csv-parse`, `csv-parser`) as potential supply chain attack surface and incorrectly proposed moving them to `devDependencies` to "reduce surface". However, if these are actually used in a production or deployed backend environment, this would cause a critical crash at runtime.
**Learning:** Security auditing tools and AI agents need full context to determine if a dependency is actually used in production vs strictly in local development before proposing movement to `devDependencies`. "Reducing attack surface" should not come at the cost of application stability. Red Team Critic personas need to be specifically trained to evaluate whether a proposed fix will break the build or runtime environment.
**Prevention:** 1. Do not move dependencies to `devDependencies` without confirming they are unused in production environments or deployed edge functions. 2. Red team agents must consider operational stability alongside theoretical security best practices.
